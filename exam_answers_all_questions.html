Question 7 Incorrect - 0.00/1.00
What is the value of the variable a after the following fragment? int a = 12, b = a / 5; if (a < 0) b += 1; a = b;
2.4
3.4
12
2

Explanation - INDENTATION TRAP!

Correct answer: 2

Step-by-step execution:

int a = 12, b = a / 5;  // a=12, b=12/5=2 (integer division)
if (a < 0)              // 12 < 0 is FALSE
    b += 1;             // NOT executed (inside if)
    a = b;              // ALWAYS executed (NOT inside if!)
                        // Result: a = b = 2

The trap: The line a = b; looks like it's inside the if block due to indentation, but there are no braces! Without braces, only the FIRST statement after if is controlled by the condition.

What actually happens:

if (a < 0) {
    b += 1;    // Only this is conditional
}
a = b;         // This ALWAYS runs!

Why you got it wrong: You likely thought a stayed 12, but a=b always executes, setting a to 2.
