Question 1 ‚úì Correct
Matching Question | 2.00 points
Match the following terms with their definitions:
Term Definition Status
Compiler Converts a program to machine language ‚úì Correct
Interpreter Converts and executes a program line by line ‚úì Correct

üìö Detailed Explanation

Compiler: A compiler is a program that translates source code written in a high-level programming language into machine language (executable code) all at once. The resulting executable can be run multiple times without recompilation.

Interpreter: An interpreter translates and executes code line by line at runtime. It doesn't produce a separate executable file but executes the code directly.

üîë Key Differences:

  • Compilation Speed: Compilers take longer initially but produce faster executables
  • Execution Speed: Compiled programs run faster than interpreted ones
  • Error Detection: Compilers detect all errors before execution; interpreters stop at the first error during execution
  • Memory Usage: Compilers generate machine code (more memory); interpreters don't create intermediate files
  • Examples: C++ uses a compiler (g++, clang), Python uses an interpreter

üìö Detailed Explanation

The printf function is a fundamental C/C++ function for formatted output.

üîë Printf Format Specifiers:

Specifier Type Example
%d or %i Integer (signed) printf("%d", 42);
%f Float/Double printf("%f", 3.14);
%c Character printf("%c", 'A');
%s String printf("%s", "Hello");
%u Unsigned integer printf("%u", 100u);
%x, %X Hexadecimal printf("%x", 255);

‚ö†Ô∏è Common Mistake:

Using %f for integers is WRONG! The %f format specifier is specifically for floating-point numbers (float and double types). Using it with integers will produce undefined behavior or garbage values.

C++ Example
// Correct usage
printf("%d %f %c %s\n", 42, 3.14, 'A', "Hello");
// Output: 42 3.140000 A Hello

// Wrong usage - DO NOT DO THIS!
int x = 42;
printf("%f", x);  // UNDEFINED BEHAVIOR!

// Correct for integer:
printf("%d", x);   // Output: 42
Question 3 ‚ö† Partially Correct (1.33/2.00)
Multiple Choice | 2.00 points
What are the differences between variable types int and unsigned?
‚úì Your Correct Selections:
  • Size in memory
  • Valid range of values
Missing Correct Answer:
  • Set of values

üìö Detailed Explanation

The difference between int (signed integer) and unsigned (unsigned integer) lies in how they represent numbers:

Property int (signed) unsigned
Size in memory Typically 4 bytes (32 bits) Typically 4 bytes (32 bits)
Range of values -2,147,483,648 to 2,147,483,647 0 to 4,294,967,295
Set of values Includes negative numbers Only non-negative numbers (0 and positive)
Sign bit Uses 1 bit for sign All bits for magnitude

üîë Key Concepts:

  • Memory Size: Both occupy the same memory (4 bytes on most systems)
  • Value Range: Different ranges due to how the sign bit is used
  • Set of Values: int can represent negative values; unsigned cannot
  • Bit representation: In int, the most significant bit (MSB) represents the sign (0 = positive, 1 = negative)
  • Overflow behavior: Different overflow behaviors when exceeding limits
C++ Example
#include <iostream>
#include <climits>
using namespace std;

int main() {
    int signedInt = -100;           // Can be negative
    unsigned unsignedInt = 100;     // Cannot be negative
    
    cout << "int range: " << INT_MIN << " to " << INT_MAX << endl;
    // Output: int range: -2147483648 to 2147483647
    
    cout << "unsigned range: 0 to " << UINT_MAX << endl;
    // Output: unsigned range: 0 to 4294967295
    
    // What happens with negative values in unsigned?
    unsigned badIdea = -1;  // Wraps around!
    cout << "unsigned = -1: " << badIdea << endl;
    // Output: unsigned = -1: 4294967295 (wraps to max value)
    
    return 0;
}

üìö Detailed Explanation

Static arrays in C++ have specific requirements and characteristics:

üîë Static Array Requirements:

  • Compile-time size: The size must be known when the program is compiled
  • Constant expression: Size must be a constant integer expression (literal, const variable, or enum)
  • Stack allocation: Local static arrays are allocated on the stack
  • Contiguous memory: Elements are stored in contiguous memory locations
  • Fixed size: Cannot be resized after declaration
C++ Examples
// ‚úì CORRECT static array declarations
int arr1[10];                    // Literal constant size
const int SIZE = 5;
int arr2[SIZE];                  // const variable size
int arr3[2 + 3];                 // Constant expression

// ‚úó WRONG - size not known at compile time
int n;
cin >> n;
int arr4[n];                     // ERROR! Variable size (except in C99/C11 VLA)

// ‚úó WRONG - non-const variable
int size = 10;
int arr5[size];                  // ERROR! Not a constant expression

// Static array characteristics:
int numbers[100];                // 100 * sizeof(int) bytes on stack
// - Allocated at compile time
// - Fixed size
// - Fast access
// - Limited by stack size

‚ö†Ô∏è Common Mistakes:

  • Using non-const variables as array size
  • Trying to resize static arrays
  • Confusing static arrays with dynamic arrays (new/delete)
  • Exceeding stack size with very large static arrays

Comparison with Dynamic Arrays:

// Static array (stack, compile-time size)
int staticArr[100];

// Dynamic array (heap, runtime size)
int size;
cin >> size;
int* dynamicArr = new int[size];
// ... use array ...
delete[] dynamicArr;  // Must free memory!
Question 5 ‚úì Correct (1.00/1.00)
Numerical Answer | 1.00 point
What value does arr[4] have in the following code? 
int arr[] = {5, 7, 3, 10, 2};
‚úì Your Answer: 10

This is correct!

üìö Detailed Explanation

Arrays in C++ use zero-based indexing, meaning the first element is at index 0.

Index 0 1 2 3 4
Value 5 7 3 10 2

Therefore, arr[4] refers to the fifth element, which has the value 10.

üîë Key Concepts - Array Indexing:

  • Zero-based indexing: First element is at index 0
  • Last element: At index (size - 1)
  • Array bounds: Valid indices are from 0 to (size-1)
  • Out of bounds: Accessing invalid indices causes undefined behavior
C++ Example
#include <iostream>
using namespace std;

int main() {
    int arr[] = {5, 7, 3, 10, 2};
    
    cout << "arr[0] = " << arr[0] << endl;   // Output: 5
    cout << "arr[1] = " << arr[1] << endl;   // Output: 7
    cout << "arr[2] = " << arr[2] << endl;   // Output: 3
    cout << "arr[3] = " << arr[3] << endl;   // Output: 10
    cout << "arr[4] = " << arr[4] << endl;   // Output: 2
    
    // Array size
    int size = sizeof(arr) / sizeof(arr[0]);
    cout << "Array size: " << size << endl;   // Output: 5
    
    // Last element
    cout << "Last element: " << arr[size-1] << endl;  // Output: 2
    
    return 0;
}

‚ö†Ô∏è Common Mistakes:

  • Thinking arrays start at index 1 (one-based indexing)
  • Accessing arr[size] instead of arr[size-1]
  • Off-by-one errors in loops
Question 6 ‚ö† Partially Correct (1.33/2.00)
Multiple Choice | 2.00 points
Which data types can be used in the expression of a switch statement?
‚úì Your Correct Selections:
  • char
  • int
Missing Correct Answer:
  • bool

üìö Detailed Explanation

The switch statement in C++ can only work with integral types and enumeration types.

üîë Valid Types for Switch Expression:

Type Category Examples Valid?
Integer types int, short, long, long long ‚úì Yes
Character types char, signed char, unsigned char ‚úì Yes
Boolean type bool ‚úì Yes
Enumeration types enum Color {RED, GREEN, BLUE} ‚úì Yes
Floating-point types float, double, long double ‚úó No
String types string, char* ‚úó No
C++ Examples
// ‚úì VALID - int
int choice = 2;
switch(choice) {
    case 1: cout << "One"; break;
    case 2: cout << "Two"; break;
    default: cout << "Other";
}

// ‚úì VALID - char
char grade = 'A';
switch(grade) {
    case 'A': cout << "Excellent"; break;
    case 'B': cout << "Good"; break;
    case 'C': cout << "Average"; break;
    default: cout << "Below average";
}

// ‚úì VALID - bool
bool isActive = true;
switch(isActive) {
    case true: cout << "Active"; break;
    case false: cout << "Inactive"; break;
}

// ‚úì VALID - enum
enum Day {MON, TUE, WED, THU, FRI};
Day today = WED;
switch(today) {
    case MON: cout << "Monday"; break;
    case TUE: cout << "Tuesday"; break;
    case WED: cout << "Wednesday"; break;
    default: cout << "Other day";
}

// ‚úó INVALID - float (compilation error!)
float value = 1.5;
switch(value) {  // ERROR!
    case 1.5: cout << "One point five"; break;
}

// ‚úó INVALID - string (compilation error!)
string name = "John";
switch(name) {  // ERROR!
    case "John": cout << "Hello John"; break;
}

‚ö†Ô∏è Why Floating-Point and Strings Don't Work:

  • Floating-point: Precision issues make equality comparisons unreliable
  • Strings: String comparison requires function calls, not simple equality
  • Performance: Switch is optimized for integer comparison (jump tables)

Alternative for strings:

// Use if-else for strings
string name = "John";
if (name == "John") {
    cout << "Hello John";
} else if (name == "Jane") {
    cout << "Hello Jane";
} else {
    cout << "Hello stranger";
}
Question 7 ‚úó Incorrect (0.00/1.00)
Multiple Choice | 1.00 point
What value does variable a have after executing this code? 
int a = 2;
if (a > 5) {
    a = 12;
}
‚úó Your Answer: 12
‚úì Correct Answer: 2

üìö Detailed Explanation

Let's trace through the code step by step:

Step-by-Step Execution
int a = 2;           // Step 1: a is initialized to 2

if (a > 5) {         // Step 2: Check if 2 > 5
                     // This is FALSE (2 is NOT greater than 5)
    a = 12;          // Step 3: This line is SKIPPED
}                    

// Final result: a = 2 (unchanged)

üîë If Statement Logic:

  • Condition evaluation: a > 5 evaluates to false (because 2 is not greater than 5)
  • Code block execution: The code inside the if block only runs when the condition is true
  • Result: Since the condition is false, the line a = 12; never executes
  • Final value: a remains 2
Complete Example with Output
#include <iostream>
using namespace std;

int main() {
    int a = 2;
    cout << "Initial value: a = " << a << endl;  // Output: a = 2
    
    if (a > 5) {
        cout << "Inside if block" << endl;  // This won't print
        a = 12;
    }
    
    cout << "Final value: a = " << a << endl;   // Output: a = 2
    
    return 0;
}

/* Program Output:
Initial value: a = 2
Final value: a = 2
*/

‚ö†Ô∏è Common Misunderstanding:

Many students incorrectly assume that the assignment a = 12 always executes. Remember:

  • The if block only runs when the condition is true
  • In this case, 2 > 5 is false
  • Therefore, a = 12 is never executed
  • The variable retains its original value: 2

What if we wanted to set a to 12?

int a = 2;
if (a <= 5) {        // Changed condition (2 <= 5 is TRUE)
    a = 12;
}
// Now a = 12

// OR use else:
int a = 2;
if (a > 5) {
    a = 12;
} else {
    a = 100;         // This executes
}
// Now a = 100
Question 8 ‚úì Correct (3.00/3.00)
Matching Question | 3.00 points
Match programming terminology with their definitions:
Term Definition Status
Semantics The meaning of syntactically correct program ‚úì Correct
Lexeme Sequence of characters having collective meaning ‚úì Correct
Alphabet Set of characters used to write programs ‚úì Correct
Expression Syntactic entity whose evaluation produces a value ‚úì Correct
Operator Symbol representing operations to be performed on operands ‚úì Correct
Syntax Rules defining valid program structure ‚úì Correct

üìö Detailed Explanation

Programming Language Concepts:

1Ô∏è‚É£ Syntax:

Definition: The set of rules that define the structure of valid programs.

  • Specifies how to write statements, declarations, expressions
  • Defines proper ordering of keywords and symbols
  • Example: if (condition) { statements } is correct syntax
  • Counter-example: if condition { } is incorrect syntax in C++

2Ô∏è‚É£ Semantics:

Definition: The meaning of syntactically correct code.

  • Defines what the code actually does
  • Example: int x = 5; means "create an integer variable named x and assign it the value 5"
  • Semantic errors: code compiles but doesn't do what you intend

3Ô∏è‚É£ Alphabet:

Definition: The set of characters that can be used in a program.

  • Letters: A-Z, a-z
  • Digits: 0-9
  • Special characters: +, -, *, /, =, {, }, etc.
  • Whitespace: space, tab, newline

4Ô∏è‚É£ Lexeme:

Definition: A sequence of characters that represents a single meaningful unit.

  • Keywords: if, while, int, return
  • Identifiers: variable names, function names
  • Literals: 42, "hello", 3.14
  • Operators: +, -, ==, &&

5Ô∏è‚É£ Expression:

Definition: A combination of values, variables, and operators that evaluates to a value.

  • Arithmetic: 2 + 3 * 4 evaluates to 14
  • Boolean: x > 5 && y < 10 evaluates to true or false
  • Assignment: x = 5 (assigns 5 to x)
  • Function call: sqrt(25) evaluates to 5

6Ô∏è‚É£ Operator:

Definition: A symbol that represents an operation to be performed.

  • Arithmetic: +, -, *, /, %
  • Comparison: ==, !=, <, >, <=, >=
  • Logical: &&, ||, !
  • Assignment: =, +=, -=
Practical Example
// SYNTAX: structure of the code
int x = 5;  // Correct syntax

// SEMANTICS: meaning of the code
// This creates an integer variable x and assigns it value 5

// ALPHABET: characters used
// i, n, t, space, x, =, 5, ;

// LEXEMES: meaningful units
// "int" (keyword), "x" (identifier), "=" (operator), "5" (literal), ";" (separator)

// EXPRESSION: evaluates to a value
x + 10     // Expression that evaluates to 15
x > 3      // Expression that evaluates to true

// OPERATORS: symbols for operations
+, -, *, /, =, ==, <, >, etc.

‚ö†Ô∏è Syntax vs Semantics Errors:

Type Example Detection
Syntax Error int x = ; (missing value) Caught by compiler
Semantic Error int age = -5; (illogical but valid) Runs but logically wrong
